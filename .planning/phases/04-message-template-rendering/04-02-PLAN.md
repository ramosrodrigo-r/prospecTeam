Create `src/utils/template.js` with the following exact implementation:

```javascript
export function renderTemplate(template, vars) {
  return template.replace(/\{\{(\w+)\}\}/g, (_, key) => {
    const val = vars[key]
    return val != null ? String(val) : ''
  })
}
```

How it works:
- Regex `/\{\{(\w+)\}\}/g` matches all `{{word}}` patterns globally (flag `g`)
- Capture group `(\w+)` extracts the variable name (letters, digits, underscore)
- `vars[key]` looks up the value in the provided object
- `val != null` checks for both `null` and `undefined` (loose equality)
- `String(val)` converts numbers (like rating 4.2) to string
- If val is null/undefined, returns empty string `''` (never `"null"` or `"undefined"`)
- Unknown placeholders (key not in vars) also resolve to `''` because `vars[key]` is `undefined`

```javascript
import { readFileSync } from 'node:fs'
import { join, dirname } from 'node:path'
import { fileURLToPath } from 'node:url'
import { renderTemplate } from '../utils/template.js'

const __dirname = dirname(fileURLToPath(import.meta.url))
const TEMPLATE_PATH = join(__dirname, '..', '..', 'templates', 'outreach.txt')

export function renderMessage(prospect, { cidade, categoria }) {
  const template = readFileSync(TEMPLATE_PATH, 'utf8')
  return renderTemplate(template, {
    nome:      prospect.name   ?? '',
    rating:    prospect.rating ?? '',
    categoria: categoria       ?? '',
    cidade:    cidade          ?? ''
  })
}
```

Key design decisions:
- `renderMessage(prospect, { cidade, categoria })` -- prospect object is kept immutable. cidade/categoria come as separate params because they are search context, not prospect attributes (per RESEARCH.md Pitfall 4)
- Path uses `import.meta.url` + `dirname` + `join` for absolute resolution -- works regardless of process cwd (per RESEARCH.md Pitfall 3)
- `TEMPLATE_PATH` navigates from `src/stages/` up 2 levels to project root, then into `templates/`
- Each `??` fallback on prospect fields prevents `String(null)` producing `"null"` literal (per RESEARCH.md Pitfall 1)
- Template is read with `readFileSync` synchronous I/O, consistent with project pattern in `history.js`
